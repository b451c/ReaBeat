ReaBeat Guide

Neural beat detection and tempo mapping for REAPER — v2.0

ReaBeat is a native C++ extension that brings state-of-the-art neural beat detection directly into REAPER. Select an audio item, click Detect, and ReaBeat identifies every beat and downbeat using the beat-this model (CPJKU, ISMIR 2024). Edit beats visually in an interactive waveform, then apply one of three action modes to match tempo, insert a tempo map, or add stretch markers.

No Python. No server. No installer. One file in your UserPlugins folder.

♫
Match Tempo
Change playrate to hit a target BPM. Pitch preserved. Align to project or a reference track.
☶
Insert Tempo Map
Make REAPER's grid follow the audio. Constant or variable tempo. No item modification.
▮
Insert Stretch Markers
Place markers at every beat or downbeat. Quantize timing to a regular grid.

Installation

ReaPack (recommended)

  1. In REAPER: Extensions > ReaPack > Import repositories...
  2. Paste: https://raw.githubusercontent.com/b451c/ReaBeat/main/index.xml
  3. Extensions > ReaPack > Browse packages, search "ReaBeat"
  4. Right-click > Install, then restart REAPER

Manual install

  1. Download the extension file for your platform from GitHub Releases:
    • macOS (Apple Silicon): reaper_reabeat-arm64.dylib
    • macOS (Intel): reaper_reabeat-x86_64.dylib
    • Windows: reaper_reabeat-x64.dll
    • Linux: reaper_reabeat-x64.so
  2. Copy to your REAPER UserPlugins folder:
    • macOS: ~/Library/Application Support/REAPER/UserPlugins/
    • Windows: %APPDATA%\REAPER\UserPlugins\
    • Linux: ~/.config/REAPER/UserPlugins/
  3. Restart REAPER
Linux users: You may need to install graphics drivers and curl if not already present:
sudo apt install mesa-utils libcurl4
mesa-utils provides the OpenGL context needed for rendering. libcurl4 is required for model auto-download. Most desktop Linux distributions have these pre-installed.

First Run

The first time you run a detection, ReaBeat downloads the neural network model (approximately 79 MB) to ~/.reabeat/models/. This is a one-time download. Subsequent detections use the cached model file.

The model is beat_this_final0.onnx from CPJKU (ISMIR 2024), which achieves state-of-the-art F1 scores for beat and downbeat detection. It runs locally on your machine using ONNX Runtime — no internet connection needed after the initial download.

GPU acceleration ReaBeat uses ONNX Runtime for inference. On machines with CUDA-capable GPUs, inference will be faster. On CPU, detection typically takes a few seconds per minute of audio.

Opening the Window

  1. Open Actions > Show action list
  2. Search for "ReaBeat"
  3. Run "ReaBeat: Show Window"
  4. Optionally, assign a keyboard shortcut to this action for quick access

ReaBeat opens as a floating, resizable window (minimum 500x300, maximum 600x900). The window stays on top of REAPER when REAPER is in focus, and moves to the normal window level when you switch to another application.

When no detection has been run, a splash screen shows the ReaBeat icon with the tagline "Neural beat detection for REAPER."

Detecting Beats

To detect beats in an audio item:

Select a media item on the REAPER timeline. ReaBeat automatically follows your selection.

Click the Detect button. ReaBeat loads the audio, computes a mel spectrogram, runs neural inference, and post-processes the results.

The waveform view populates with detected beats (gold lines) and downbeats (taller gold lines with bar numbers). The detected BPM and time signature appear in the header.

Detection runs on a background thread, so REAPER stays responsive. A progress indicator shows the current stage. The pipeline includes:

  • Mel spectrogram — audio resampled to 22050 Hz, 128-band mel filterbank
  • Neural inference — ONNX model processes overlapping chunks of audio
  • Peak detection — separate processing for beat and downbeat heads
  • Gap filling — missing beats interpolated using sub-threshold model hints
  • Onset refinement — each beat snapped to the nearest audio transient within +/-30ms
  • Tempo estimation — phase-aware regression for accurate BPM
  • Downbeat cleaning — neural downbeats validated and gaps filled
Multi-item cache Detection results are cached per item GUID. Switch to a different item, detect it, then switch back — your previous results are still there. The status shows "(cached)" for previously detected items.

Waveform Display

The main area shows a mirrored RMS waveform of the audio. When zoomed in, individual peaks are interpolated for smooth rendering.

  • Gold vertical lines — detected beat positions
  • Taller gold lines with bar numbers — downbeat positions (first beat of each bar)
  • Dim teal lines — REAPER stretch markers (if any exist), shown as reference in beat mode
  • Time ruler — bottom strip with auto-scaling time intervals
  • White crosshair — semi-transparent vertical line following the mouse cursor
  • Gold scroll bar — appears at the bottom when zoomed in, shows current position
  • Mode badge — top-right corner shows "BEATS" (gray) or "MARKERS" (gold)

The waveform height scales dynamically to fill the space between the header and the controls area. Click anywhere on the waveform to seek (set the REAPER edit cursor to that position).

Confidence and BPM

After detection, the header displays:

  • BPM value — the detected tempo, clickable to edit manually
  • Confidence percentage — color-coded: green above 80%, gold between 50-80%, red below 50%
  • Time signature — detected from downbeat spacing (typically 4/4 or 3/4)
  • Beat count — total number of detected beats
  • Beat flash indicator — gold dot near the BPM that pulses on each beat during playback (larger pulse on downbeats)

If the filename contains a BPM value that differs more than 5% from the detected tempo, a hint displays "(filename: X BPM)" to flag the discrepancy.

Click the BPM value to type in a custom tempo. This only affects the Match Tempo rate calculation — beat and downbeat positions are absolute seconds and are never recalculated from the tempo. After editing, "(was X)" shows the original detected value.

Beat Editing

BEATS MODE

After detection, the waveform enters beat editing mode. Every beat and downbeat is displayed as a gold vertical line. You can move, add, delete, and promote/demote beats before applying an action. All edits modify the internal detection data — nothing touches REAPER until you click Apply.

An internal undo stack (up to 50 states) lets you freely experiment with beat positions. Use Cmd+Z to undo and Cmd+Shift+Z to redo. This undo system is separate from REAPER's.

Interactions

Move a beat

Hover near a beat line to see a glow halo and drag handle. The cursor changes to a left-right arrow. Click and drag horizontally to reposition the beat. A dashed ghost line shows the original position as reference. The drag activates after 3 pixels of movement to prevent accidental edits.

Add a beat

Double-click on an empty area of the waveform (away from existing beats). A new beat is inserted at that position, automatically snapped to the nearest audio transient within +/-30ms using RMS peak detection.

Toggle downbeat

Double-click directly on an existing beat line to toggle its downbeat status. A regular beat becomes a downbeat (bar boundary), and a downbeat becomes a regular beat. Downbeats are displayed taller with bar numbers.

Delete a beat

Right-click on a beat line to remove it. The surrounding beats are unaffected.

Jump to next gap

Press N to scroll and zoom to the next gap in the beat grid — any interval longer than 1.35 times the median beat interval. The view wraps around to the beginning after reaching the end of the audio.

Gap Highlighting

After detection, the waveform highlights intervals where beats are likely missing:

  • Red tint background — covers any interval longer than 1.35 times the median beat interval
  • Dashed suggestion lines — appear at RMS peak positions (strongest transients) within each gap. Larger gaps show multiple suggestion lines (a 3x gap shows 2 suggestion lines, etc.)

These visual cues help you find and fix missing beats quickly. Double-click on a suggestion line position to add the missing beat.

Match Tempo

Adjusts the item's playrate so it plays back at a target BPM. Pitch is preserved using REAPER's elastique pitch shifting. The original audio file is not modified.

Match to Project

The simplest workflow. ReaBeat reads your project tempo and calculates the playrate needed to match it. Click Apply and the item's playrate changes. The item plays at the project BPM while maintaining its original pitch.

Match to Reference

The "Match to:" dropdown lists the project tempo plus all other detected items in the session. Select a reference item, and Apply triggers a full synchronization pipeline: tempo map from the reference, playrate adjustment, downbeat alignment, and stretch markers on both tracks.

See the Multi-Track Sync section for the complete workflow.

BPM Controls

  • [/2] button — halve the displayed BPM (useful when detection doubles the tempo)
  • [x2] button — double the displayed BPM (useful when detection halves the tempo)
  • "Set session" button — sets the REAPER project tempo to the detected BPM
  • Align first downbeat — shifts the item so the first detected downbeat lands on a bar line
Octave correction The [/2] and [x2] buttons change only the displayed tempo for rate calculation. Beat and downbeat positions are absolute seconds and are never affected. Internally, detected BPM is constrained to 78-185 BPM (optimized for modern music including 140 BPM trap, while filtering 200+ BPM artifacts).

Insert Tempo Map

Syncs REAPER's grid to the audio without modifying the item. After applying, REAPER's bars and beats align with the detected beat positions. This is ideal for working with recordings that have natural tempo variations.

Three sub-modes

Sub-modeBehaviorBest for
ConstantSingle BPM for the entire itemStudio recordings with steady tempo
Variable - barsBPM recalculated at each downbeatLive recordings with gradual drift
Variable - beatsBPM recalculated at each beatRubato performances, extreme variation

Tempo maps include octave correction and BPM filtering to prevent artifacts from false detections.

Non-destructive Insert Tempo Map only changes REAPER's tempo envelope. The audio item is untouched. You can undo the entire operation with Cmd+Z.

Insert Stretch Markers

Places REAPER stretch markers at detected beat positions (every beat or downbeats only). Stretch markers warp the audio's internal timing so beats land on a regular grid.

Scope

  • Every beat — a stretch marker at each detected beat position
  • Downbeats only — markers only at bar boundaries (fewer markers, less warping)

Quantize modes

ModeBehaviorUse case
OffMarkers placed at exact detected positions. No time warping.Visual reference only
Constant tempoDownbeat subdivision with internal grid. Each bar is mathematically divided into equal beats.Single track, regular beat grid
Session gridDownbeat subdivision + snap to REAPER's project grid. Markers align with the session tempo.Multi-track sync, matching to session

Stretch quality

Choose the time-stretch algorithm quality for the markers:

  • Balanced — general purpose, good for most material
  • Transient — preserves attack quality, best for drums and percussion
  • Tonal — preserves pitch quality, best for sustained notes and vocals
False-positive filtering Before creating stretch markers, ReaBeat automatically removes beats that are less than half a beat interval apart. This prevents erroneous double-beat detections from creating extreme stretch ratios.

Marker Edit Mode

MARKERS MODE

After you click Apply with stretch markers (or multi-track sync), the waveform switches to marker edit mode. The mode badge in the top-right changes from "BEATS" to "MARKERS" with a gold border.

In this mode, every edit goes directly to the REAPER API. Each operation is wrapped in an undo block, so Cmd+Z uses REAPER's undo system (not the internal beat undo stack). The original neural beat positions remain visible as dim gray reference lines behind the gold marker lines.

ReaBeat periodically re-reads stretch markers from REAPER (approximately once per second), so external edits made in REAPER's arrange view are reflected automatically.

Interactions

Add a marker

Double-click on an empty area of the waveform. A new stretch marker is inserted at that position, snapped to the nearest RMS peak within +/-30ms. The destination position is interpolated from the adjacent markers to preserve the local stretch rate. A gold flash ring provides visual feedback.

Move a marker

Click and drag an existing marker. The stretch ratios between adjacent markers update live during the drag, so you can see the effect in real time. Release to commit the change.

Delete a marker

Right-click on a marker to remove it. A gold flash ring confirms the deletion. All other markers are unaffected.

Stretch Ratios

Between each pair of adjacent markers, a stretch ratio label shows how much the audio is being warped. The labels are color-coded:

  • Green (0.93x - 1.07x) — minimal stretching, nearly transparent to the listener
  • Gold (0.70x - 1.30x) — moderate stretching, generally acceptable
  • Red (outside 0.70x - 1.30x) — heavy stretching, may produce audible artifacts

A beat grid overlay of dim teal dashed lines shows the expected beat positions based on the detected BPM. This helps you see where markers should ideally land versus where they currently are.

Mode transitions Applying Stretch Markers or Multi-Track Sync exits and re-enters marker mode (fresh markers). Applying Tempo Map or Match Tempo (project) preserves the current marker mode. Running a new detection exits marker mode and returns to beat editing.

Multi-Track Sync

ReaBeat can synchronize two independently recorded tracks to the same beat grid. This is the most powerful workflow: one click aligns both tracks to a common tempo map with stretch markers.

Step-by-step workflow

Detect the reference track. Select the item you want to use as the timing reference (typically the most rhythmically stable recording — drums, click track, etc.). Click Detect.

Detect the slave track. Select the item you want to synchronize to the reference. Click Detect. Both items now have cached detection results.

Select the reference. In the "Match to:" dropdown, select the reference item. It appears by name alongside the project tempo option.

Click Apply. ReaBeat executes the full synchronization pipeline automatically.

What happens on Apply

The sync pipeline performs six operations in sequence, all wrapped in a single REAPER undo block:

  1. Find the reference item by its GUID in the project
  2. Insert a tempo map from the reference (REAPER's grid aligns to the reference's first downbeat and BPM)
  3. Stretch markers on the reference with Session grid quantize (regularizes the reference to its own grid)
  4. Match the slave's playrate to the reference BPM (pitch preserved)
  5. Align the slave's downbeat (shift so the first downbeat matches the reference's first downbeat)
  6. Stretch markers on the slave with Session grid quantize (snaps to the same REAPER grid)
Both tracks get stretch markers This is essential for accurate sync. Without stretch markers on the reference, its natural micro-timing variations would cause drift relative to the grid-locked slave track. Stretching both to the same grid eliminates this.

Tips and Workflows

Regularize tempo for mixing

Detect the audio, choose Insert Stretch Markers with Constant tempo quantize, and click Apply. Every beat snaps to a mathematically regular grid. Use marker edit mode to fix any problem spots — double-click to add missing markers, right-click to remove bad ones.

Make REAPER's grid follow your audio

Detect the audio, choose Insert Tempo Map with Variable - bars sub-mode, and click Apply. REAPER's bars and beats now follow the natural tempo of the recording. This is ideal for adding click tracks, quantizing MIDI, or aligning overdubs.

Match to session tempo

If your session is at 120 BPM and you have a loop recorded at 124 BPM, select Match Tempo with "Match to: Project (120 BPM)" and click Apply. The item's playrate changes to slow it down to 120 BPM with pitch preserved.

Multi-track sync

You have a guitar recorded in one take and drums in another, both with natural tempo fluctuations. Detect both items. On the guitar item, open the "Match to:" dropdown, select the drums as reference, and Apply. Both tracks get stretch markers on the same grid, perfectly in sync.

Manual correction in marker mode

After applying stretch markers, you notice one beat sounds wrong. In marker edit mode, drag the offending marker to a better position (watch the stretch ratio labels turn green). Or right-click to delete it entirely if it is a false positive. Each edit is a single REAPER undo point.

Wrong BPM detected

If ReaBeat detects 140 BPM but the track is really 70 BPM (or vice versa), click the [/2] or [x2] button next to the BPM display. This corrects the octave without re-running detection. You can also click the BPM value to type in any tempo manually.

Editing beats before applying

If the neural detection missed some beats or placed them slightly off, fix them in beat editing mode before applying. Press N to jump to each gap. Double-click to add missing beats, drag to reposition. The suggestion lines (dashed) show where transients are strongest.

Troubleshooting

Low confidence score

A confidence below 70% usually means the audio has weak rhythmic content (ambient, spoken word, rubato classical). Detection results may be unreliable. Try editing beats manually in beat mode before applying.

Wrong BPM (double or half)

This is normal for certain genres. Use the [/2] or [x2] buttons to correct octave errors. The BPM value only affects tempo matching and rate calculations — beat positions remain unchanged.

No beats detected

Ensure the selected item contains audio (not an empty item or MIDI). Check that the model file exists at ~/.reabeat/models/beat_this_final0.onnx. If it is missing, delete the models directory and re-run detection to trigger a fresh download.

Model download fails

If the automatic download fails (firewall, network issues), you can manually download the ONNX model from the GitHub Releases page and place it at ~/.reabeat/models/beat_this_final0.onnx.

Red stretch ratios after applying

Red ratio labels (outside 0.70x - 1.30x) indicate heavy time-stretching that will likely produce audible artifacts. This usually means a false detection or missing beat. In marker edit mode, right-click the problematic marker to delete it, or double-click nearby to add a missing marker that distributes the stretching more evenly.

Metronome not aligned

After inserting a tempo map, toggle the REAPER metronome with the metronome button in the ReaBeat header (gold when active). If the click does not align, try the Variable - bars sub-mode for more precise grid matching.
